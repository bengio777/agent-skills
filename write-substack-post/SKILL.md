# BG : Media Post-Coaching

Coach the author through transforming raw media (video, photo, audio) into a tight, engaging, multimedia-rich Substack post (~5 minute read) that blends storytelling with insight. Use when the author says "write a substack post", "I have media I want to post", "let's write a post", or provides media with the intent to create content.

## Workflow

### Step 1: Select Source Media

Ask: "What media are we working with today? Drop a file, paste a YouTube/Spotify link, or tell me what you're working with."

Two paths:
- **Author already knows** — They provide file(s) or link(s). Confirm selection and proceed.
- **Author is browsing** — Coach them: "Look for media that hits you in one of three ways: an emotional reaction, a story behind it, or strong visual quality. What catches your eye?"

Accept any combination of:
- **Local files** (photos, short clips) → analyzed directly
- **YouTube links** (videos) → Substack auto-embeds these
- **Spotify links** (audio/music) → Substack auto-embeds these
- **iCloud references** → author provides the file or a compressed version

For each piece, note the source type — this determines how it gets embedded in the final post. Confirm before moving on.

### Step 2: Extract the Story

Use the **Media Analysis** skill (`skills/media-analysis/SKILL.md`) to:
1. Automatically analyze the media (transcribe, extract metadata, describe visuals)
2. Present 2-3 story angles for the author to choose from

If no angles resonate, ask: "What's the story behind this? What would you tell a friend about this moment?"

### Step 3: Coach the Story Structure

Use the **Story Coaching** skill (`skills/story-coaching/SKILL.md`) to walk the author through building the narrative using the Amuse-Bouche framework:
1. Anchor → 2. Essential Details → 3. Dismount → 4. Hook (last)

At every stage: propose, ask the author to react, confirm before moving on.

### Step 4/5: Draft the Post with Integrated Multimedia

Use the **Draft & Multimedia** skill (`skills/draft-multimedia/SKILL.md`) to:
1. Map media to story beats
2. Generate the full draft (~1,200-1,500 words) with media embedded
3. Present for author review

Ask: "What feels right? What feels off? Where does it not sound like you?"

### Step 6: Review and Tighten

Use the **Editorial Coaching** skill (`skills/editorial-coaching/SKILL.md`) to:
1. Flag bloat, check hook and dismount, trim to length
2. Explain the "why" behind every edit suggestion
3. Let the author accept, reject, or modify each one

When the author says it's ready, move to formatting.

### Step 7: Format for Substack

Prepare the post for manual publishing:
1. Format as clean Substack-ready content (headings, paragraphs, media placement markers)
2. Write a subtitle recommendation
3. Suggest tags/categories
4. Generate `media-sources.md` listing all media with source type and location:
   - Local files → `media/filename.jpg` (in post repo)
   - YouTube videos → full URL
   - Spotify audio → full URL
   - iCloud originals → iCloud path reference
5. Present the publishing checklist:
   - Copy content into Substack editor
   - Upload local photos at marked positions
   - Paste YouTube/Spotify links where indicated (Substack auto-embeds)
   - Add subtitle and preview text
   - Set tags
   - Preview and publish

Ask: "Publish now or schedule for later?"

### Step 8: Post-Publish Reflection

After the author confirms the post is live:
1. "What felt easy?"
2. "Where did you get stuck?"
3. "What would you steal from this post for next time?"
4. "Confidence rating: 1-5?"

Log the reflection. If post 3+, surface patterns from previous reflections. After post 5, offer voice calibration.

## Guidelines

- Ask one question at a time — never present a wall of text
- Always propose first, then ask the author to react
- Keep the coaching conversational, not academic
- Protect the author's authentic voice — never rewrite their personality
- Enforce brevity at every stage — "an amuse-bouche, not a 7-course meal"
- The goal is to teach the craft, not just produce the post
